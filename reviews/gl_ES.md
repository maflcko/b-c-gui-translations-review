# Translations Review by LLM (✨ experimental)

The review quality depends on the LLM and the language. Currently, a fast LLM without rate limits is used. If you are interested in better quality for a specific language, please file an issue to ask for it to be re-run with a stronger model.

[If the result is of low quality, please file an issue to find a better LLM for this language.](../../issues/new?title=%5Bgl_ES%5D%20low%20quality)

        <source>Choose the address to receive coins with</source>
        <translation>Selecciona o enderezo do que recibir moedas</translation>
    
YES, the translation is problematic. The verb "receive" has been mistranslated as "do que recibir" (of which to receive), which is grammatically incorrect. The preposition "with" in the source text implies the source of the coins, not something to receive them *from*.

```
        <source>The old passphrase entered for the wallet decryption is incorrect. It contains a null character (ie - a zero byte). If the passphrase was set with a version of this software prior to 25.0, please try again with only the characters up to — but not including — the first null character.</source>
        <translation>Produciuse un erro non cambio de frase de contrasinal</translation>
    
YES, The translation is problematic. The Spanish translation is completely different from the English source text and does not convey the original meaning. Additionally, it contains a meaningless phrase that suggests the translation is autogenerated or nonsensical.

Correct translation:
A vosa frase de contrasinal antiga para o descifrado da carteira é incorrecta. Contén un carácter nulo (é dicir, un byte cero). Se a frase de contrasinal se configurou cunha versión deste software anterior á 25.0, inténteo de novo só cos caracteres ata — pero sen incluír — o primeiro carácter nulo.
```
